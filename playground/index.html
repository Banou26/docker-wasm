Dockerfile → wasm

Drop a Dockerfile. No backend. The build runs inside an alpine+buildah image c2w'd to wasm, in your browser tab, with network egress over @webvpn through this repo's c2w-webvpn netstack.

How this works
  1. Your Dockerfile is base64-encoded into the URL hash (#dockerfile=…); it never leaves the browser.
  2. We navigate to the c2w-webvpn runtime page, pointing it at a single pre-built playground.wasm (≈150 MB, alpine + buildah, c2w'd once).
  3. The wasm boots inside the browser. When the shell prompt appears, the page auto-types a build script: decode the Dockerfile from the hash, write to /work/Dockerfile, run buildah --storage-driver vfs bud --isolation chroot, then buildah run --tty to drop you into the built container.
  4. Buildah's network (registry pulls, RUN-step apk/apt) flows through this repo's gVisor netstack proxy, over @webvpn, out to the real internet. No CORS, real DNS, real TCP/UDP.

First Bochs boot takes a few minutes (full x86 emulation). Subsequent builds inside the same tab are fast.
