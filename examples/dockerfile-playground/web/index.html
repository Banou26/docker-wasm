            How this works
            
    
                
  1. Frontend POSTs your Dockerfile text to /api/build.
    2. 
                
  2. Backend writes it to a tmp dir and runs docker build, then c2w <tag> out.wasm.
    3. 
                
  3. Build log is streamed back over SSE.
    4. 
                
  4. On success, the run link opens /?net=webvpn&wasm=<jobId> — the same
                    c2w-webvpn runtime that ships in examples/alpine-curl/, but pointed at your fresh wasm.
    5. 
                
  5. Egress goes guest → gVisor netstack → @webvpn → real internet. No CORS limits, real DNS,
                    real TCP/UDP.
    6. 
            

            
Bochs is the bottleneck: even a small FROM alpine takes 5–10 minutes to boot
                under x86 emulation. c2w itself takes ~5 min for the first build of a fresh
                image (later builds reuse buildkit cache).
